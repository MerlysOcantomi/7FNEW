feat(today): support moving work between user and ai

Adds the My work / AI work split to /today and the compact bottom
drawer, with Send to AI and Take over row actions backed by a new
PATCH /api/tasks/[id]/assignee endpoint. Ownership is driven entirely
by WorkspaceTask.assigneeType — no schema changes — and proposed AI
rows surface a read-only "Proposed" pill since Approve/Dismiss stays
in Inbox / Smart Hub for now.

- modules/today/types.ts: extend TodayItem with assigneeType + isProposed
- modules/today/aggregator.ts: widen WorkspaceTask filter to include
  status="proposed" and forward the new fields per row
- modules/today/lanes.ts: pure getTodayLane + splitBucketsByLane helpers
- modules/tasks/service.ts: add updateWorkspaceTaskAssignment writing
  only assigneeType + assigneeId (no status/executionMode/suggestedBy)
- app/api/tasks/[id]/assignee/route.ts: new PATCH with body { to }
- lib/today/lane-client.ts: sendToAI / takeOver client helpers
- components/today: responsive two-column layout on /today, stacked
  compact lanes in the bottom drawer, lane-aware row actions with
  toast-on-error and refetch-on-success
