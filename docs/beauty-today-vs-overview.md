# Beauty Today vs Beauty Overview

## Current decision

The current Beauty/Finesse surface lives in `/today`.

It should be understood as **Beauty Today / Finesse Today**, not as a separate strategic Overview page.

For Beauty, Today currently acts as the operational home of the day:
- current and upcoming appointments
- free appointment slots
- confirmations
- client care signals
- Finesse guidance
- immediate decisions
- one small visual/content opportunity if useful

## Naming note

Some legacy/prototype references may use the word “overview” because the design handoff was named “Finesse Overview”.

In product architecture, this is still `/today`.

Do not create a second Beauty Overview that duplicates the same agenda-first surface.

## Future Beauty Overview

A future Beauty Overview, if built, should be more strategic and should not compete with Today.

It should focus on:
- weekly/monthly business health
- booking trends
- retention
- revenue and payments
- marketing/content performance
- inventory signals
- longer-term opportunities and risks

## Boundary

Beauty Today answers:
“What needs attention today?”

Beauty Overview should answer:
“How is the beauty business performing overall?”
